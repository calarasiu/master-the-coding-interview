Graphs

  • A set of values connected in a pair wise fashion - each node/ vertex is connected to other nodes through a so called edge
  • used to represent connections - think of maps for example - amazon uses it for their recommendation engines/ g maps to detect the shortest path to your destination
  • trees/linked lists are graphs

  • Graphs properties

  • 1 a) directed graphs - -> / <- / <-> graphs whose edges have a direction one way or double direction - twitter uses directed graphs for the connections as you can follow someone but the followed person doesn't has to follow you back -> to pay attention when this are cyclic or acyclic graphs
  • 1 b) undirected graphs - 0-0-0 - graphs that have just double direction edges - facebook uses it for the friendship connections - is mutual - the person you are firend with needs to be friend with you as well
  • 2 a) weighted graphs - have information on the edges not just on nodes - for example the distance between 2 nodes
  • 2 b) unweighted graph - hold info just in their nodes
  • 3 a) cyclic graphs - the nodes are connected in a circular fashion - you have the possibility to go back to the start point(it doesn't need to be cyclic the whole graph but at least 3 nodes ) - maps
  • 3 b) acyclic graphs - you cannot go back to the starting point -> goes in one direction -
  • DAG - directed acyclic graph - used by ioda blockchain website
