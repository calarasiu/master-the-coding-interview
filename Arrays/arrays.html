Array is the data structure that has the smallest footprint

if all you need is to store some data and iterate through it - array is the way to go

Type

  • Static arrays you need to know from the declaration how much space it will take
  • Dynamic arrays - the type of arrays used by the high level languages -
    • you don't need to worry about the space to allocate in memory as the system will take care of it
    • sometimes adding element to this arrays will cost you O(n) time as the arrays needs to be copied at another location in memory to be able to fit the extra element

Big O for the most common operations

  • Lookup - O(1) - constant time - if we have the index of the item
  • Push - O(1) - just adds an element at the end of the array
  • Pop - O(1) - deletes an element from the end
  • Unshift - (O(n)) - adds an element at the beginning of the array => it needs to shift the indexes for all the items of the array
  • Splice(startPoint, nItemsToDelete, eToAdd) - (O(n)) - worst case scenario - it needs to shift indexes, same principle as Unshift
